## 新增了什么

### 选项A：search 工具
- 函数：`search(keyword, path=".")` —— 递归遍历目录，逐行搜索关键词
- 跳过二进制文件（.png/.exe/.dll 等）和无用目录（.git/__pycache__/logs）
- 最多返回 50 条命中，防止结果太长撑爆上下文（和 c3/c4 的上下文管理思路一致）
- 搜不到时返回友好提示"没有找到包含「xxx」的文件"
- 在 SYSTEM 提示和 TOOLS 字典里都注册了，三件套齐全

### 选项B：代码审查员
- 改了 `subagent()` 的 system 消息：从"你是子助手"改成"你是代码审查员"
- 审查员的人设：读代码、找逻辑漏洞（空输入、越界、异常没处理）、只报告不修改
- 子 agent 配了 `read_file` 和 `bash` 两个工具（原来只有 bash）
- 主 SYSTEM 里说明了"用户需要检查代码质量时派审查员"
- 验收：让它审查 demo_project，能定位到 analyze.py 的 summary() 在无标签时会 IndexError

### 选项C：防崩加固
- 洞1（未知工具名）：派发前先 `if name not in TOOLS`，不在就提示"没有叫「xxx」的工具，可用的有：…"
- 洞2（无限重试）：加 `retries` 计数器，连续失败 ≥ `MAX_RETRIES`(5) 次 `break`
- 关键细节：成功一步后 `retries = 0` 清零，否则正常长任务会被误杀
- "解析失败"和"未知工具"两种错给 AI 不同的提示，它才知道怎么改

### Capstone：修 bug
- bug：`analyze.py` 第 19 行 `counts[0][0]`，当无标签笔记时 `counts` 为空列表 → IndexError
- 修复：加 `if not counts:` 空检查，返回"还没有任何标签"
- 详见 `capstone_bugfix.md`

## 遇到的最大一个坑

**模型不一定乖乖输出纯 JSON。** kimi-k2 有时会用 ` ```json ` 包裹回复，或者在 JSON 前后加解释文字。c0 直接 `json.loads(reply)` 经常崩。

解决方案（c1 开始的 `parse()` 函数）：
1. 先 `strip()` 去首尾空白
2. `strip("`")` 去掉 markdown 代码块的反引号
3. `removeprefix("json")` 去掉 `json` 标记
4. `s[s.find("{"): s.rfind("}") + 1]` 只取第一个 `{` 到最后一个 `}` 之间的内容

这套容错解析从 c1 一直用到 c5，是整个 mini Claude Code 能跑起来的基础。
